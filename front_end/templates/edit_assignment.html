Custom scoring:
By default, the assignment score is calculated as the average of the exercise scores within the assignment. If you desire, you can specify custom scoring logic. For example, suppose an assignment has five exercises. An instructor might specify that if a student successfully completes one exercise (20%), the student receives a passing grade (60%). And/or they might specify that if a student completes all but one of the exercises (80%), the student receives full points (100%) for the assignment. Please indicate your preferences below.
Provide one or more scoring ranges and an adjusted score for each range. If the sum of a student's exercise scores fall within a given threshold, they will receive the adjusted score. If a student earns a score that is not covered by one of the specified ranges, their score will not be adjusted. All ranges and adjusted scores must be specified as percentages (example: 90). For the highest scoring range, the upper end of the scoring range is inclusive; for all others, it is not.

Require security codes to access this assignment:
You can require that each student provide a security code before they access the assignment. Typically, this option is used when you want to ensure students complete an assignment in a particular physical location (often used for exams or quizzes). Additionally, when they complete the assignment, a confirmation code can be provided. You can require that students provide this code as they leave the physical location where they completed the assignment (exam/quiz). After an assignment is created, you can generate security codes or view students' security codes.
